### 1단계: Dense & Sparse 하이브리드 검색 (Hybrid Search)
단일 임베딩 모델에만 의존하는 방식에서 벗어나, 문맥을 파악하는 **Dense Retrieval**과 키워드를 매칭하는 **Sparse Retrieval(BM25)**을 결합합니다.
* **Dense (Embedding):** 문장의 전반적인 의미와 동의어를 포착 (예: '맞춤법'과 '정형 규칙'의 연관성 파악)
* **Sparse (BM25):** 문장 구조가 다르더라도 질의에 포함된 고유명사나 핵심 단어(`되`, `돼` 등)가 박혀있다면 높은 가중치로 추출
* **앙상블 기법:** 두 검색 엔진의 결과를 **RRF(Reciprocal Rank Fusion)** 알고리즘으로 결합하여, 유저가 어떤 형태의 문장을 입력하더라도 정답 조각이 무조건 1차 후보군(Top-20) 안에 포함되도록 보장합니다.

### 2단계: 크로스 인코더 기반 리랭킹 (Cross-Encoder Re-ranking)
1차 하이브리드 검색으로 추출된 상위 후보군들을 대상으로 질문과의 연관성을 정밀 재검사합니다.
* **메커니즘:** Bi-Encoder(기존 임베딩) 방식은 질문과 문서를 독립적으로 벡터화하여 연산하므로 비대칭 문장 구조에서 점수가 낮게 나오지만, Cross-Encoder(Re-ranker)는 **[질문 + 문서]를 하나의 쌍으로 묶어 딥러닝 모델에 통째로 입력**합니다.
* **효과:** 구조가 다른 문장이라도 실제 정답일 확률을 문맥적으로 교차 검증하므로, 0.4 대에 머물던 신뢰도 점수가 **0.8~0.9 대의 명확한 수치로 보정**됩니다. 백엔드는 이 리랭킹 점수를 기준으로 최종 컨텍스트를 필터링합니다.

### 3단계: 인덱싱 타임 데이터 증강 (Indexing-time Augmentation)
데이터를 VectorDB에 저장하기 전, 전처리 단계에서 텍스트에 맥락을 인위적으로 주입하는 전략입니다.

1. **앤스로픽 스타일 '콘텍스추얼 RAG' (Contextual RAG):**
   문서를 자르기 전, LLM을 활용해 각 청크가 전체 문서에서 가지는 위치와 배경을 요약하여 청크 앞부분에 꼬리표(Prefix)로 붙여줍니다.
   * *기존 Chunk:* `... 마케팅 비용은 50억 원으로 15% 증가했습니다. ...`
   * *개선 Chunk:* `[이 조각은 A기업 2026년 1부기 실적서의 비용 분석 파트입니다] ... 마케팅 비용은 50억 원으로 15% 증가했습니다. ...`

2. **가상 질문 생성 (Hypothetical Questions):**
   각 단락/퀴즈 데이터를 인덱싱할 때, LLM에게 *"사용자들이 이 정보를 찾기 위해 던질법한 예상 질문 3가지를 생성해줘"*라고 요청한 뒤, 이 예상 질문들을 청크 데이터와 함께 임베딩합니다. 사용자의 예측 불가능한 질문이 내가 심어놓은 '예상 질문 벡터'와 결합하면서 검색 성공률이 극대화됩니다.

3. **요약본 검색 ➡️ 원문 전달 패턴:**
   데이터베이스에는 가볍고 핵심만 담긴 **[요약문 + 소제목]**을 인덱싱하여 검색 속도와 키워드 매칭 정확도를 높이고, 검색이 완료되어 LLM에게 컨텍스트를 던질 때는 **[원본 단락 전체]**를 팩킹하여 전달합니다. (도서관에서 목차로 책을 찾고 읽을 때는 본문을 읽는 원리)
