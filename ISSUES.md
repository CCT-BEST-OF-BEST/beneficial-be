## 🔴 Issue 1: 비대칭 검색 (Asymmetric Retrieval) 문제

### 증상
```
쿼리:  "되와 돼의 차이가 뭐야?"         유사도: 0.39  ❌
쿼리:  "가르치다 가르키다 차이"          유사도: 0.65  ✅
쿼리:  "맞추다 맞히다 뭐가 다르지"      유사도: 0.66  ✅
```

### 원인
- `jhgan/ko-sroberta-multitask`는 **STS(Semantic Textual Similarity)** 모델
- 두 문장이 같은 의미일 때 높은 점수 → 문장 쌍(paraphrase) 탐지에 최적화
- RAG는 **비대칭 검색**: 질문(구어체) ↔ 문서(정의/설명 형태) → 형태가 달라 유사도 낮음

```
질문: "되와 돼의 차이가 뭐야?"
문서: "단어: 되다/돼다 의미: 어떤 상태가 이루어지다..."
→ 의미는 관련있지만 형태가 달라 0.39에 그침
```

- 단어를 **그대로 포함한 쿼리**는 잘 됨 (토큰 레벨 매칭)
- **구어체 / 형태소 변형** 쿼리는 실패 ("되다" → "되", "돼다" → "돼")

## 🔴 Issue 2: PDF 비관련 문서가 상위 랭크 차지

### 증상
```
쿼리: "되와 돼의 차이가 뭐야?"

검색 결과:
  1위: [pdf] 제52항: 한자어 발음 규칙...  유사도: 0.46  ← 완전 비관련
  2위: [pdf] 제24항: -거리다 용언...      유사도: 0.44  ← 완전 비관련
  3위: [kwp] 그렇게 하면 안 ( ). 정답: 돼  유사도: 0.43
```

### 원인
- PDF 79개 청크 중 한국어 공문서 텍스트들이 구조적으로 유사한 벡터를 가짐
- Dense 검색만으로는 키워드 명중 없이 구조적 유사성으로 오검색 발생

## 🟡 Issue 4: 유사도 임계값 기준 불명확

| 모델 | 임계값 | 비고 |
|---|---|---|
| text-embedding-ada-002 | 0.6 | 0.63~0.66 → 대부분 통과 |
| ko-sroberta (현재) | 0.3 | 최고값이 0.46 수준 |

- 임계값이 너무 낮으면 비관련 문서가 컨텍스트에 포함됨
- 임계값이 너무 높으면 아무것도 검색 안 됨
- 현재는 0.3으로 낮춰놨으나, 비관련 문서가 컨텍스트에 들어오는 문제 존재

## 🟢 개선 계획 (RAG.md 기반)

### 1순위: 인덱싱 타임 가상 질문 생성

각 문서를 인덱싱할 때 GPT로 예상 질문을 생성해서 함께 저장

```
기존 문서: "단어: 되다/돼다 의미: 어떤 상태가 이루어지다..."

가상 질문 3개 생성:
  - "되와 돼의 차이가 뭐야?"
  - "안돼 안되 어느 게 맞아?"
  - "돼가 맞는 표현이야 되가 맞는 표현이야?"

→ 원본 문서 + 가상 질문을 함께 임베딩하여 저장
```

**효과**: 학생이 어떤 형태로 질문해도 가상 질문 벡터와 매칭 → 유사도 0.8+ 기대

### 2순위: BM25 하이브리드 검색 + RRF

```
Dense (ko-sroberta): 의미적 유사도
Sparse (BM25):       키워드 명중 ("되", "돼" 토큰 직접 매칭)
결합 (RRF):          두 점수를 역순위 융합으로 합산
```

### 3순위: Cross-Encoder 리랭킹

- 1차 검색 결과 top-20 후보군을 (질문, 문서) 쌍으로 재평가
- `bge-reranker-large` 등 리랭커 모델 사용
- 0.4대 유사도 → 0.8~0.9대로 보정
